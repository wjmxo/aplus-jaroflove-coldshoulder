# AGENTS.md — Amazon Premium A+ Repository Router

本仓库用于制作可上传 Amazon Premium A+ 后台的女装模块图。所有任务必须先判断属于素材生产还是页面制作,再路由到对应仓库级技能。

## 1. 强制路由规则

- 素材生成 / 素材建库 / 老产品补货 / 生成模特、细节、场景、配饰、尺码标注图的任务,必须调用 `$aplus-asset-production`。
- 页面制作 / HTML 排版 / 模块重排 / 连续画布切片 / PNG 导出 / 视觉自检 / 推送成品的任务,必须调用 `$aplus-page-production`。
- 素材任务完成后必须停止,提交素材清单和缩略拼图,等待用户验收;严禁顺手进入页面制作。
- 页面制作任务只允许使用已验收素材库;素材不足时输出“补货申请清单”并暂停对应模块,不得自行生成新素材。
- 若用户同时要求素材与页面,先执行素材生产并停止等待确认;除非用户明确说明当前批次仍沿用旧流程。

## 2. Always-On 合规红线

**禁止出现:**
- 价格、折扣、促销词(free, sale, cheap, discount, deal)
- 保证/绝对化用语(best, #1, guarantee, 100%, perfect, top-rated)
- 卖家信息、联系方式、外部链接、二维码
- 评论引用、明星代言、奖项(除非能提供证明)
- 与详情页信息矛盾的内容;时效性内容(now, new, latest 谨慎使用)
- 商标符号使用不当;竞品品牌名

**写作要求:**
- 标题 ≤ 8 个单词,动词开头或利益点开头
- 正文短句,一个模块只讲一个核心信息
- 卖点写“对买家的好处”,不是罗列功能(Feature → Benefit)
- Q&A 优先回应差评中的高频疑虑
- 美国站用美式拼写

## 3. Always-On 视觉与素材红线

- `reference/` 只学习布局、配色逻辑、排版节奏、叙事结构;禁止复制参考品牌 Logo、商标、文案原句、产品图、模特照片或场景图。
- 生成照片时必须把 `assets/` 中对应的产品实拍图(`color-*.jpg`)作为参考图输入,确保领口开衩、两颗纽扣、罗纹质感、袖长与修身版型不得漂移。
- `assets/color-*.jpg` 仅作为服装还原基准,只有两个例外可直接入画:色卡模块的颜色贴纸、尺码表核对基准;其余画面一律使用 `assets/gen/` 生成素材。
- 严禁用 CSS/SVG 绘制产品插画代替产品照片。功能图标属于 UI 元素,允许用 SVG 线性圆圈图标绘制,但不得画成产品照片替代物。
- 所需照片在 `assets/` 或 `assets/gen/` 中不存在时,用灰色占位框标注“待素材”,并在交付说明中列出缺失清单。
- 不确定的品牌信息、素材、合规边界必须先问,不得编造。

## 4. Always-On 尺寸与切片纪律

- Premium A+ 主图模块默认画布为 `1464 x 600`;连续通版必须从最终尺寸开始设计,再精确切片。
- 01-03 通版如采用连续画布,尺寸为 `1464 x 1800`,切为 3 张 `1464 x 600`。
- 04-05 通版如采用连续画布,尺寸为 `1464 x 1200`,切为 2 张 `1464 x 600`。
- 切片线安全距:人脸、标题文字、纽扣特写、手部距切片线 ≥60px;躯干、腿部、背景允许且鼓励跨线。
- 切片后必须验证切片与通版逐像素一致,并导出切片线上下检查带。
- 2 倍尺寸 PNG 是最终上传成品;HTML 是唯一源文件。

## 5. 关键决策确认机制

以下决策属于用户保留决策,执行前必须列出 2-3 个方案(每个附一句理由),停下等用户确认,不得自行默认:

1. 每张通版的主推颜色(锚点+主角用色)及各颜色画面权重分配
2. 模块顺序或结构的任何调整
3. 文案卖点取舍与主标题措辞的大改
4. 删除或替换用户曾明确确认过的元素

除上述四类外的执行细节(素材姿势、装饰位置、字号微调)无需询问,按 `STYLE.md` 自行决定。

## 6. 输出清单

- `plan.md` — 模块策划方案(已经用户确认)
- `copy.md` — 全部文案 + 每图 Alt 关键词
- `modules/*.html` — 各模块设计源文件
- `output/*.png` — 2 倍尺寸成品图
- `README-upload.md` — 上传指引:每张图对应后台哪个模块、填哪些文字字段
